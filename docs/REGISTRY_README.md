# Проверка реестра увольнений/перемещений

Скрипт читает Excel-файл(ы) реестра, по каждому сотруднику проверяет в A-Tracker наличие техники и при наличии отправляет оповещение в группу. **Один и тот же файл не обрабатывается дважды** — используется учёт по пути и дате изменения (mtime).

## Куда класть файлы

1. **Папка `data/` (рекомендуется)**  
   В `config.py` задайте папку:
   ```python
   REGISTRY_FILE_PATH = "data/"
   ```
   Кладите в неё реестры с **любыми именами** (например, `Реестр ОК 17.01.2026.xls`, `Реестр ОК 01.02.2026.xls`). При каждом запуске обрабатываются только **новые** файлы (.xls и .xlsx). После обработки файл перемещается в `data/processed/` с датой в имени (например, `2026-02-02_12-30_Реестр ОК 17.01.2026.xls`).

2. **Один конкретный файл**  
   Можно указать путь к одному файлу:
   ```python
   REGISTRY_FILE_PATH = "data/Реестр ОК 17.01.2026.xls"
   ```
   Тогда обрабатывается только он (и только если ещё не был обработан).

## Как не обрабатывать один файл дважды

- **Файл состояния** `data/registry_processed.json` хранит для каждого обработанного файла путь и время изменения (mtime). Если при следующем запуске путь и mtime совпадают — файл пропускается.
- **Перемещение в `data/processed/`** — после успешной обработки файл переносится в эту папку, так что в `data/` остаются только ещё не обработанные. Отключить перемещение: в `config.py` задать `REGISTRY_PROCESSED_DIR = None`.
- Имя Excel-файла может быть любым; важен только путь и факт, что файл уже был обработан (по state или по переносу в processed).

## Формат файла

- Формат: **.xls** (Excel 97–2003) или **.xlsx**.
- На каждом листе должны быть колонки (заголовок в первой строке или в первых 10 строках):
  - **Дата приказа** — дата увольнения/перемещения
  - **Сотрудник** — ФИО (по нему идёт поиск в A-Tracker)
  - **Документ**
  - **Должность**
  - **Примечание**
- Учитываются только строки, где заполнено поле «Сотрудник».

## Запуск

```bash
# Установить зависимости (если ещё не ставили)
pip install -r requirements.txt

# Запуск: обрабатываются файлы из config.REGISTRY_FILE_PATH (папка или один файл)
python run_registry_check.py

# Или указать папку/файл явно
python run_registry_check.py "data/"
python run_registry_check.py "data/Реестр ОК 01.02.2026.xls"
```

## Поведение

- По каждой строке реестра берётся ФИО из колонки «Сотрудник».
- В A-Tracker запрашиваются активы по этому ФИО (тот же сервис, что и в боте).
- Оповещения отправляются **только по строкам, где дата уже наступила** (дата ≤ сегодня). Это можно отключить в `run_registry_check.py` (переменная `ONLY_PAST_OR_TODAY = False`).
- Если у сотрудника есть техника — в группу уходит сообщение с листом, датой, ФИО, документом, должностью, примечанием и списком техники из A-Tracker.
- Группа для оповещений: `REGISTRY_NOTIFY_GROUP_ID` в `config.py`; если не задано — используется `REPORT_GROUP_ID`.

## Запуск по расписанию (cron)

Пример: каждый день в 8:00 из папки проекта:

```bash
0 8 * * * cd /путь/к/tg_scan && /путь/к/python run_registry_check.py
```

Или через виртуальное окружение:

```bash
0 8 * * * cd /путь/к/tg_scan && .venv/bin/python run_registry_check.py
```

## Зависимости

- **.xls**: пакет `xlrd` версии **&lt; 2** (в `requirements.txt` указано `xlrd>=1.2.0,<2.0`).
- **.xlsx**: пакет `openpyxl`.
