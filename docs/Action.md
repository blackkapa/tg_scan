

7.4 Дизайнер действий (Мастера, скрипты, отчеты)
Системный объект «Действие» используется для настройки следующей логики работы:
- Настройка действий с типом «Мастер»:
a. Создание мастера;
b. Создание отчета.
- Настройка действий с типом «Скрипт»:
a. Создание уведомлений;
b. Настройка проверок корректности записей таблиц при обновлении/добавлении.
Все пользовательские скрипты могут быть использованы только в процессе выполнения
Действия. Для доступа к списку Действий нужно отрыть раздел меню «Администрирование
## \ Действия»
7.4.1 Описание полей объекта Действие
- Системное имя (только для чтения);
- Переопределенное действие - выпадающий список не коробочных действий для замены
вызова по ID или системному имени (поле доступно для действий, идущих из коробки);
- ID - ID записи;
- Название - текст;
- Тип действия – выпадающий список (Мастер или Скрипт);
- Системная  таблица – выпадающий  список  всех  таблиц  Системы для  выбора
контекста;
- Кнопка  на  форме  объекта – выпадающий  список,  определяющий  необходимость
использования  кнопки  на  форме  объекта  для  запуска  действия  (Не  требуется  /
Требуется).  Если  требуется,  обязательно  заполнить Название  кнопки и Условие
отображения;
- Код - непосредственный код мастера или скрипта;
- Расчет ответственного – код выбора ответственного (для мастера, участвующего в
рабочих процессах);
- Расчет заголовка задачи - код расчета заголовка Задачи (для мастера, участвующего в
рабочих процессах);

7.4.3 Действие с типом «Мастер»
Мастера предназначены для выполнения последовательного заранее предопределенного
набора действий (шагов), на каждом из которых возможна проверка введенных данных, а
также  выполняют  действия  с  данными.  Мастера  помогают  пользователю  выполнять
стандартные  процедуры, где заранее определена  последовательность  пользовательских
действий с данными.
В рамках выполнения Действия с типом «Мастер» происходит вывод формы мастера.
Кнопка или ссылка для отображения мастера может появляться в интерфейсе системы в
следующих случаях:
1). При обработке шага рабочего процесса, когда тип Действия этого шага = мастер;
- На  получателя  Действия  назначается  задача  по  выполнению  мастера.  На  главной
странице приложения она отображается в перечне задач.
- После выполнения мастера рабочий процесс переходит на следующий шаг.

2). При доступе из меню, когда пункт меню запускает мастер;
3). При доступе с формы объекта, когда на форме объекта есть кнопка для запуска мастера.
- Для отображения кнопки на форме объекта должно выполниться условие «Условие
отображения» на вкладке «Мастер», а также для данной роли должны быть выданы
права на доступ к данному мастеру;
- Название кнопки подставляется из поля «Текст на кнопке» Действия.
Открытие мастера в новом или текущем окне зависит от чек-бокса «Открывать в новой
вкладке» Действия.
Примечание: для Действий,  идущих  «из  коробки»,  доступно  поле  «Переопределенное
действие». Если в выпадающем списке выбрано действие, то это действие будет вызываться
во  всех  местах,  где  вызывалось  коробочное  действие  (меню,  ссылки  виджетов,  рабочие
процессы). Роли пользователей, имеющие доступ к коробочному Действию, автоматически
получат аналогичные права доступа к переопределенному Действию.

7.5 Настройка мастеров
Описание мастера представляет собой структурированный текст в формате XML, который
хранится в поле «Код» объекта Действие. Код можно редактировать в специализированном
редакторе, открывающемся по кнопке «Открыть в редакторе» (не доступна для мастеров,
идущих  «из  коробки»).  Редактор  откроется  в  соседнем  окне.  Можно  выбрать, какой  код
загрузить в редактор: из БД / локальную копию / несохраненную копию.
По кнопке «Открыть мастер» произойдет возврат на форму мастера.

Рисунок 4 – Редактор кода мастера в A-Tracker.
В качестве элементов описания мастера выступают следующие объекты:
- Страницы мастера;
- Элементы страниц мастера для отображения на форме (block и element);
- Переходы между страницами мастера;
- Пользовательские скрипты.
Комбинация данных элементов позволяет построить полноценный процесс выполнения
мастера.  Ниже  приведено  описание  общей  структуры  мастера  и  подробное  описание  его
элементов.


7.5.1 Общая структура мастера
- Мастер  в  качестве  параметров  имеет  название  title  и  указание  на  действие  по
завершению finishAction (необязательный атрибут, используется, в основном, в отчетах, когда
не требуется показывать страницу «мастер выполнен»).
- Мастер  состоит  из  страниц  <page>  каждая  из  которых  имеет  уникальное  название
SQLName, номер шага (если есть разные сценарии и необходимо создать несколько страниц
на шаге 2, например) и название страницы title.
- Внутри каждой страницы <page> располагаются блоки:
- onLoadScript – скрипт, который будет запущен перед отрисовкой страницы. Содержит
инструкции по заполнению полей на странице и динамических списков;
- buttons – блок с кнопками;
- block – блок для HTML элементов на странице. В мастере отображается как единая
строка с элементами интерфейса, таблицами или другими элементами формы мастера;
- pageTransition – скрипт, выполняемый при переходе на следующую страницу. Обычно
содержит  инструкции  по  проверке  введенных  данных  на  странице  и  выполняемые
инструкции по обновлению данных. Состоит из блоков:
o validationScript – скрипт проверки введенных значений;
o transitionScript – скрипт расчета следующего шага.

- Весь код скриптов должен быть отображен в виде:  <![CDATA[ ... ]]>.

- Порядок отрисовки мастера на странице:
- Отрисовка идет построчно.
o <block> — это одна строка;
o <element> — это элемент строки с различной шириной, кратной 1\12.
- Исключение: кнопки должны идти в блоке <buttons>, который трансформируется в
отдельную в строку, содержащую только кнопки.
- Если в редакторе кода в пустом месте написать <wizard, то будет предложено выбрать
готовую структуру мастера.

## Замечания:
- В  разметке XML мастера  все  атрибуты  необязательные,  они  по  умолчанию
заполняются негативным значением.
- Все SQLName должны быть уникальными. Невозможно на первой странице сделать
поле txtName, а на второй такое же поле txtName, чтобы оно заполнилось. Необходимо
сделать txtName2 и в onLoadScript прописать заполнение этого поля.
- finishAction="redirect to main page" - необязательный  атрибут, указывает,  что  по
завершении мастера не требуется показывать страницу «Мастер выполнен», а сразу
перенаправлять пользователя на главную страницу системы.  Атрибут подходит для
создания отчётов, когда по кнопке «Закрыть» не предполагается никаких действий.
- Все  булевы  атрибуты  (например,  readonly="1")  могут  заполняться  значениями
0/1/false/true, где 0 == false,a 1 == true.

7.5.4 Элементы блока <element>
В Системе могут использоваться следующие типы элементов блока <element>:
- Простые элементы – поля с разными типами данных для ввода информации:
o Time – строка «Время» вида “HH:mm:ss”;
o Date – строка «Дата» вида “dd:MM:yyyy”;
o Datetime – строка «Дата и время» вида “dd:MM:yyyy HH:mm:ss”;
o Bool – булево значение (0 / 1 / false / true);
o Money – денежное поле;
o Link – ссылочное поле;
o String – однострочный текст (строка);
o Multistring – многострочный текст;
o Int – целое число;
o File – загрузка файла;
o Combobox – выпадающий список значений (комбобокс);
- Сложные элементы – списки, таблицы, статический и динамический HTML-код:
o List – список / таблица для вывода данных в табличной форме;
o Progress bar – статусная строка;
o HTML – блок HTML, содержащий статический код HTML / JavaScript;
o Widget – блок, содержащий любой динамический код.

Простые элементы блока <element>.
Простые элементы содержат следующий набор полей:
- Type – тип поля, определяет какую информацию в него можно записать или показать;
- Width – ширина поля на странице, где 12 – полностью занята вся строка, а 1 – это 1\12
ширины страницы;
- SQLName – уникальное имя;
- Readonly – признак, что поле доступно только на чтение (1|0|true|false);
- Label – имя, отображаемое рядом с полем;
- Value – значение;
- Особенные поля для некоторых типов:
o link-table – ссылка на таблицу;
o filter – фильтр в ссылочной таблице.
Примеры простых элементов с различными типами:
<element type="time" width="3" SQLName="" readonly="0" label="" value=""></element>

параметра с таким именем метод вернёт false.

3). InstanceConfig.DeleteConfig("Название параметра") - удаляет сохраненный параметр.
В случае отсутствия параметра с таким названием вызовет System.Exception: Config not
found.

7.5.7 Советы при разработке мастера в части разметки
- Чтобы воспользоваться редактором кода:
- Выберите в меню «Администрирование \ Действия», выберите «Новая запись»;
- Нажмите кнопку «Создать» с пустым кодом действия.
- Нажмите кнопку «Открыть в редакторе». Открытое окно редактора автоматически
сохраняется в хранилище браузера каждые 10 секунд. Это сделано для того, чтобы
можно было просто писать, как в блокноте, не нажимая сохранить (редактор не даст
сохранить код в БД Системы с ошибками).
- Сохраните код в базе данных по «Ctrl+S».

- В момент сохранения делается резервная копия предыдущей версии. Такая реализация
необходима  для  случаев,  когда  извне  (например,  с  другой  инсталляции  Системы)
Администратор переносит код мастера и возникает ошибка, то есть произошло копирование
кода не в то действие в Систему. В этом случае существует возможность восстановления
предыдущей версии (перед сохранением), и необходимо, обновив окно редактора, выбрать,
какую версию открыть: из БД, локальную, или предыдущую.


- Весь код пишется с помощью снипетов. То есть в пустом редакторе необходимо
начать  набирать  "<",  и  будут  предложены  все  сниппеты  для  XML  структуры.  Например,
"<wizard" развернет стандартный шаблон для одностраничного мастера. В сниппетах курсор
ставится сразу туда, где нужно что-то изменить (например, у мастера это Имя мастера, потом
Имя страницы и т.д.), перемещение между этими местами для заполнения - по клавише tab.

-  В  самом  редакторе  слева  у  номера  строки  может  появляться  ошибка  разметки
(например, найден открывающийся тэг и не найден его закрывающий тэг).

- При заполнении, у элементов SQLName появляется автоматическое заполнение всех
полей всех таблиц. Поля в мастере, для удобства,  необходимо называть также,  как и поля в
объекте. Например, если необходимо на форме мастера выбрать «Заявителя» для заявки, но
Администратор  системы не помнит, как именно пишется поле lRequesterId, можно начать
писать «lreue» (с  опечатками,  в  нижнем  регистре),  и  редактор  кода  сразу  предложит
правильное название.
- Для ссылочных элементов есть атрибут link-table, в котором необходимо выбрать
таблицу, ссылкой на которую данный элемент является.
<element type="link" width="3" SQLName="" readonly="0" label="" link-table=""
filter="[seStatus] = 1" value=""></element>


- Для  упрощения  разработки, при  заполнении данного  атрибута,  редактор  кода  авто
заполняет названия всех таблиц системы. Достаточно написать «reql», чтобы получить
в выборе «itamReqLines».
- Атрибут filter="[seStatus]  =  3" не  обязательный.  С  его  помощью  можно  наложить
фильтр  для  выбираемых  записей.  Например,  когда  необходимо  дать  возможность
выбрать только Склады из всей таблицы Местоположения. Или только не уволенных
сотрудников из справочника Сотрудники.

7.6 Формирование документов по шаблону
## 7.6.1 Шаблон (.docx)
Система  позволяет  формировать  различные  документы  .docx (например,  форму  МБ-2)
внутри  мастеров  (см.    Приложение  1).  Шаблоном  документа  является  обычный  файл  с
расширением docx,  который  предварительно  заполнен  Администратором  системы  и  с
указанными метками для последующей замены внутри мастера.
Для  расстановки  меток  (Элементов  управления  содержимым)  в  документе docx.
используется вкладка "Разработчик" на ленте редактора Microsoft  Word. Если в редакторе
Word не отображается вкладка "Разработчик", ее необходимо включить, выбрав пункт меню
"Файл" -> "Параметры" -> "Настройки ленты" и установив флажок напротив "Разработчик".
Для  размещения  метки  необходимо  разместить  курсор  там, где  вы  хотите  разместить
элемент  управления,  после  чего  выбрать  в разделе  "Элементы  управления" на  вкладке
"Разработчик» тип элемента управления, который необходимо добавить (например, флажок,
текстовое поле, список и т. д.).
После добавления элемента управления можно настроить его свойства, например, указать
подпись для поля.
Подход к созданию шаблона:
1). Создать / открыть любой docx файл.
2). Отформатировать файл при необходимости (шрифты, таблицы, выделения цветом).
3). Установить метки в местах будущей замены. Например, если сформирован   файл, у
которого в Заголовке должно быть написано «Накладная No <порядковый_номер> от
<сегодняшняя_дата>», то этом случае:
a. вместо номера и даты требуется установить метки;
b. после замены метки форматирование текста сохранится, как и у слова-метки.

## 7.6.2 Шаблон (.xlsx)
Система позволяет формировать различные документы .xlsx (например, счет на оплату)
внутри мастеров. Шаблоном документа является обычный xlsx файл, который предварительно
заполнен  Администратором  системы  и  с  указанными  метками  для  последующей  замены
внутри мастера.
Для расстановки меток в документе xslx. используется вкладка "Диспетчер имен" на ленте
редактора Microsoft Excel. Если в редакторе Excel не отображается вкладка "Диспетчер имен",
ее необходимо включить, выбрав пункт меню "Файл" -> "Параметры" -> "Настройки ленты" и
установив флажок напротив "Диспетчер имен".
Для размещения метки необходимо сначала создать диапазон ячеек в «Диспетчере имен»,
содержащий ячейки, в которые планируется добавлять метки. Далее в каждую целевую ячейку
необходимо добавить конструкцию {{item.TagName}}, где вместо TagName нужно указать
имя добавляемой метки.
Подход к созданию шаблона:
1). Создать/открыть любой xlsx файл;
2). Отформатировать файл при необходимости (шрифты, таблицы, выделения цветом);
3). На вкладке "Диспетчер имен" запустить одноименный инструмент «Диспетчер имен»;
4). В  окне  инструмента "Диспетчер  имен" создать новый  диапазон  ячеек,  содержащий
места будущих замен;
5). Установить метки в местах будущей замены. Например, если сформирован файл, у
которого в Заголовке должно быть написано «Накладная No <порядковый_номер> от
<сегодняшняя_дата>», то в этом случае:
o вместо номера и даты установить метки;
o после замены метки форматирование ячейки сохранится как в шаблоне.
